## ğŸ› ï¸ Available Tools

### Flutter Project Health Audit
Complete auditing system for Flutter/Dart projects that includes:

- **Automated analysis**: 9 analysis categories with scoring
- **Multi-app support**: Handles simple and monorepo repository structures
- **Standardized reports**: 16-section format ready for Google Docs
- **CI/CD integration**: GitHub Actions workflow analysis
- **Test coverage**: Execution, aggregation, and 70% minimum threshold validation
- **ChatGPT integration**: Specialized prompt for generating executive summaries

**Location**: `flutter_project_health_audit/`

**Documentation**: See `flutter_project_health_audit/README.md` for detailed instructions.

### Flutter Best Practices Check
Specialized tool for validating code quality and adherence to specific development standards:

- **Micro-level analysis**: Evaluation of testing quality, architecture compliance, and code standards
- **Live Standards Validation**: Validates code against live `.mdc` standards from GitHub
- **Report Template**: Standardized plain-text reporting
- **Modular Rules**: Independent analysis rules for Testing, Architecture, and Code Standards

**Location**: `flutter_best_practices_check/`

**Documentation**: See `flutter_best_practices_check/README.md` for detailed instructions.

## ğŸš€ Quick Start

### For Flutter projects

#### Option 1: Health Audit Only (Macro-level)
```bash
# Navigate to the health audit tool
cd flutter_project_health_audit/

# Execute the plan
@flutter-health.plan.md

# See flutter_project_health_audit/README.md for detailed instructions
```

#### Option 2: Combined Audit (Macro + Micro-level)
```bash
# Step 1: Execute Health Audit
@flutter_project_health_audit/plan/flutter-health.plan.md

# Step 2: After Health Audit completes, you will be prompted to execute Best Practices Check
# The system will ask if you want to run it - you must confirm before execution
@flutter_best_practices_check/plan/best_practices.plan.md
```

**Important**: The Best Practices Check is **NEVER executed automatically**. After the Health Audit completes, you will be prompted to confirm if you want to run it.

**What You Get**:
- **Health Audit**: Infrastructure, CI/CD, security, documentation analysis
- **Best Practices Check**: Code quality, testing standards, architecture compliance (optional, user-confirmed)

**Reports Generated**:
- Health Audit: `./reports/flutter_audit.txt`
- Best Practices: Generated by the best practices generator (see plan for location)
